# Resume variants

Drop one or more resume variants here as markdown (e.g. `generalist.md`, or by focus like `clinical.md` / `management.md`). Apply will recommend **which variant to send** for a given job and **which bullets to swap** to match the posting — it won't fabricate, only re-emphasize what's already true.

## Why variants?

If you target meaningfully different kinds of role (e.g. a clinical role vs a management role, or "design" vs "research"), a tailored base resume per type beats one generic resume. If you only target one kind of role, a single variant is fine.

## Format

Plain markdown, one file per variant. A workable structure:

```markdown
# <Variant name — e.g. "Generalist," "Management track," "Design-focused">

## SUMMARY
<2–3 lines. Lead with the identity this variant sells.>

## EXPERIENCE

### <Company> | <location> | <dates>
**<Title>**
- <Achievement bullet — action + outcome + metric>
- <...>

## SKILLS & TOOLS
**<Category>:** <skills>

## EDUCATION
<degree, school, year — optional>
```

> Keep the markdown as the source of truth; export to PDF when you actually apply. PDFs are gitignored by default (they're binaries and often contain the most PII).
